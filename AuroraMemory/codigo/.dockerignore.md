# .dockerignore

**Responsabilidade:** Define arquivos e diretórios a serem ignorados durante a construção da imagem Docker, otimizando o build e evitando vazamento de dados sensíveis.

**Itens ignorados:**
- `node_modules` — dependências locais (serão reinstaladas no container)
- `.next` — build local do Next.js
- `.git` — histórico de versionamento
- `.env` / `.env.local` — variáveis de ambiente sensíveis
- `*.md` — arquivos de documentação
- `.gitignore` — arquivo de configuração do Git

**Relações:** Complementa o `Dockerfile` ao garantir que apenas o código-fonte necessário seja copiado para a imagem, reduzindo seu tamanho e tempo de build.
