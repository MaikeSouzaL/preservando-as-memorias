# `.vercelignore`

Arquivo de configuração `.vercelignore` para o deploy na Vercel.

**Responsabilidade:** Define quais arquivos e diretórios devem ser **excluídos** do deploy na plataforma Vercel, evitando que artefatos locais, dependências e arquivos temporários sejam enviados para produção.

**Itens ignorados:**
- `node_modules` — dependências instaladas localmente (a Vercel as reinstala)
- `.next` — build local do Next.js (a Vercel gera o próprio build)
- `.vercel` — configurações locais do projeto Vercel
- `public/uploads` — uploads de usuário (não versionados)
- `test-results` — saída de testes
- `tsconfig.tsbuildinfo` — cache do TypeScript
- `*.log` — logs de execução

**Relação:** Complementa o `vercel.json` (configuração de deploy) e o `next.config.js` (configuração do framework), garantindo que apenas o código-fonte e assets necessários sejam enviados para o ambiente de produção.
